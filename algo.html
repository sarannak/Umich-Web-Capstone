What is an Algorithm?

An algorithm is a finite sequence of instructions/steps to solve a problem or perform a specific task.A computer program can be viewed as an elaborate algorithm. In mathematics and computer science, an algorithm usually means a small procedure that solves a recurrent problem. Algorithms have the following characteristics: 1)Input(zero or more) 2)Output(1 or more) 3)Finiteness 4)Unambiguous.

Analysis of Algorithm

Suppose a specific problem has more than one solution. In order to find the optimal algorithm for that problem analysis of algorithm is used. Algorithms are analysed on how they depend on their input size meaning how the algorithm will perform for large and small input size. Generally, there are two parameters for analysis of an algorithm: 1)How much space it takes 2)How much time will it take.

Space Complexity

Space Complexity is the amount of space needed by an algorithm. An efficient algorithm takes space as small as possible. In general total space = fixed space requirement + variable space requirement. Nowadays space is not the concern as we have huge amount of memory available.

Time Complexity

Time Complexity is time required by an algorithm to solve a problem. Now the running time of a particular program can be different on different machines so we consider the fact how well the algorithm performs as input size is increased. Algorithms are function of their input size of f(n) ,where n is the input size. Practically we see that there is a trade off between space and time i.e if we increase space complexity time complexity decreases and vice-versa. There are 3 cases of running time of algorithm: 1)Best Case 2)Average Case 3)Worst Case.

Big O Notation

It is mathematical notation used to denote the time/space complexity. It is an upper bound for an algorithm running time. O(1) means no matter what the input size is the algorithm will finish its task in a constant amount of time depending only on the speed of the machine. Whereas O(n) means as we increase input size running time increases linearly. In case of O(n^2) as input is increased slightly running time increases manyfolds. So O(n^2)>O(n)>O(1).

Searching

Searching refers to the opreation of finding the location of a desired key element from a set of records, data structure, library, etc.. There are many kinds of searching techniques (google - searching algorithms) of which the most simple is Linear Search(O(n)) i.e we search one by one for a desired key in a given dataset. In case of binary search(O(logn)) we can rapidly find an element from a sorted order of data elements.

Sorting

Sorting refers to operation of rearranging the data elements in a given order. Just as searching there are many kinds of sorting algorithms. By using analysis we can compare the different sorting algorithms and can effectively select which is best as per the given requirements. Bubble Sort is the most simple sorting algorithm which checks consective elements and swaps them. It traverses the data set more than 1 times and in each pass it places the greatest element(in case of ascending order) then 2nd greatest and so on in the rightmost place.
